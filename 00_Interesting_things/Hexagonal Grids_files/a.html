                              I modified and simplified Charles Chamber's code, such that it will work with Axial coordinates as proscribed here.

                              The division by the sqrt(3) was superfluous, if you're saving the hex radius, widths, and heights somewhere (which, for my project, I already was ...).

                              Here's working code (screen x, z coordinates start at upper left of the screen):

                              /**
                              * Convert a screen point to a hex coordinate
                              */
                              public Coord PointToCoord(double x, double z) {
                              x = (x - halfHexWidth) / hexWidth;

                              double t1 = z / hexRadius, t2 = Math.Floor(x + t1);
                              double r = Math.Floor((Math.Floor(t1 - x) + t2) / 3);
                              double q = Math.Floor((Math.Floor(2 * x + 1) + t2) / 3) - r;

                              return new Coord((int) q, (int) r);
                              }

                              For Hexagon shaped grids, one needs to offset x, z by the center of the 0, 0 hex:

                              if (shape == Shape.Hexagon) {
                              x -= grid.gridRadius * hexWidth;
                              z -= grid.gridRadius * (qrtrHexHeight + halfHexHeight);
                              }

                              Otherwise the above PointToCoord method will correctly produce the same coordinates as shown on the images in the "map based storage" section above.

                              For further efficiency, and since these are doubles, it would even be possible to completely remove the Math.Floor calls, and change them to bitwise operations if one so desired.
